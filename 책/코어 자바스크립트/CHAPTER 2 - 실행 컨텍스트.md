## 실행 컨텍스트

- 실행 컨텍스트란?
  - **“실행할 코드에 제공할 환경 정보들을 모아놓은 객체”**
  - JS의 동적 언어로서의 성격을 가장 잘 파악할 수 있는 개념
- 실행 컨텍스트 동작
  - 동일한 환경에 있는 코드들을 실행할 때 필요한 환경 정보들을 모아 컨텍스트를 구성
  - 컨텍스트를 콜 스택(call stack)에 쌓아올림
  - 가장 위에 쌓여있는 컨텍스트와 관련 있는 코드들을 실행하는 식으로 전체 코드의 환경과 순서를 보장
- 동일한 환경(하나의 실행 컨텍스트)을 구성하는 방법
  - 전역공간, `eval()` 함수, 함수
  - `eval`은 인자로 받은 코드를 실행해서 악의적으로 사용할 수 있기 때문에 사용하면 안됨
  - 전역도 컨트롤 할 수 없으므로 일반적으로 실행 컨텍스트를 구성하는 방법은 **함수를 실행** 하는 것 뿐이다.
- 전역 컨텍스트
  - 일반적인 실행 컨텍스트와 다를 것이 없다.
  - 최상단 공간은 코드 내부에서 별도의 실행 명령이 없어도 브라우저에서 자동으로 실행하므로, 자바스크립트 파일이 열리는 순간 전역 컨텍스트가 활성화 된다.
- 실행 컨텍스트의 실행 시점
  - 스택 구조 상 실행 컨텍스트가 콜 스택의 맨 위에 쌓이는 순간이 곧 현재 실행할 코드에 관여하게 되는 시점이다.
  - 기존 컨텍스트는 새로 쌓인 컨텍스트보다 아래에 위치할 수밖에 없기 때문이다.
  - 실행 컨텍스트가 활성화될 때 자바스크립트 엔진은 해당 컨텍스트에 관련된 코드들을 실행하는 데 필요한 환경 정보들을 수집해서 실행 컨텍스트 객체에 저장한다.

### 실행 컨텍스트의 담기는 정보

- 변수 환경(VariableEnvironment)
  - `var`로 선언된 변수, 함수 선언문, 초기값 설정
  - 현재 컨텍스트 내의 식별자들에 대한 정보 + 외부 환경 정보
  - 선언 시점의 렉시컬환경의 스냅샷으로, 변경사항은 반영되지 않음
  - 호이스팅이 발생하는 공간
- 렉시컬 환경(Lexical Environment)
  - 코드 실행 중 계속 변화함
  - 클로저의 핵심
  - 구성
    - Environment Record
      - `let`, `const` 변수
      - 함수 내부 변수
      - 매개변수
    - Outer Environment Reference
      - 상위 스코프를 가리킴 (스코프 체인)
- This Binding
  - 현재 실행 컨텍스트에서의 `this` 값
  - 상황별
    - 전역: `window` (브라우저)
    - 함수 호출: `undefined` (strict mode)
    - 메서드 호출: 해당 객체
    - 생성자: 새로 생성된 인스턴스

## EnvironmentRecord와 호이스팅

- EnvironmentRecord에 저장되는 현재 컨텍스트와 관련된 코드의 식별자 정보들 컨텍스트 내부 전체를 처음부터 끝까지 훑어나가며 **‘순서대로’** 수집됨
  - 컨텍스트를 구성하는 함수에 지정된 매개변수 식별자
  - 선언 함수가 있을 경우 그 함수 자체
  - `var`로 선언된 변수의 식별자 등
- 코드가 실행되기 점임에도 자바스크립트 엔진은 이미 해당 환경에 속한 코드의 변수명들을 모두 알고 있다.
  - 엔진의 실제 동작 방식 대신 ‘자바스크립트 엔진은 식별자들을 최상단으로 끌어올려놓은 다음 실제 코드를 실행한다’ 라고 쉽게 생각할 수 있다.
  - 호이스팅은 ‘끌어 올린다’라고 해석할 수 있으며, 변수 정보를 수집하는 과정을 더욱 이해하기 쉬운 방법으로 대체한 가상의 개념이다.

### 함수 선언문과 표현식

- 함수 선언문
  - `function` 정의부만 존재하고 별도의 할당 명령이 없는 것
  - 반드시 함수명이 정의돼 있어햐 한다.
- 함수 표현식
  - `function`을 별도의 변수에 할당하는 것
  - 함수명이 있을 필요는 없다.

```jsx
// 함수 선언문, 함수명 a가 곧 변수명
function a() {
  /* ... */
}
a(); // ok

// (익명) 함수 표현식, 변수명 b가 곧 함수명
var b = function () {
  /* ... */
};
b(); // ok

// 기명 함수 표현식, 변수명은 c, 함수명은 d
var c = function d() {
  /* ... */
};
c(); // ok
d(); // error
```

- 기명 함수 표현식의 특이점
  - 함수명은 오직 내부에서만 접근할 수 있다.
  - 익명 함수 표현식은 과거 브라우저에서 함수명이 undefined 또는 unnamed로 나왔다.
- 함수 선언문의 차이점
  - 재선언이 가능하다.
    - 호이스팅으로 인해 마지막 선언한 함수로만 실행된다.
  - 호이스팅이되므로 선언부 이전에 사용할 수 있다.

## 스코프, 스코프 체인, OuterEnvironmentReference

- 스코프(scope)
  - 식별자에 대한 유효범위이다.
  - **ES5 까지의 JS는 함수에서만 스코프가 생성되었다.**
- 스코프 체인
  - ‘식별자의 유효범위’를 안에서부터 바깥으로 차례로 검색해나가는 것
  - OuterEnvironmentReference로 인해 스코프 체인이 가능하다.
  - OuterEnvironmentReference는 현재 호출된 함수가 선언될 당시의 LexicalEnvironment를 참조한다.
  - **무조건 스코프 체인 상에서 가장 먼저 발견된 식별자에만 접근 가능**하다.
- **변수 은닉화**(variable shadowing)
  - inner 함수 내부에서 a 변수를 선언하면 전역 공간에서 선언한 동일한 이름의 a 변수에 접근할 수 없다.
