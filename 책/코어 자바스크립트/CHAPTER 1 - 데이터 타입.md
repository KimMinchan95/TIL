## 데이터 타입의 종류

- Primitive type (원시형)
  - number, string, boolean, null, undefined, Symbol(ES6)
- Reference type (참조형)
  - object, Array, Function, Date, RexExp(정규표현식), Map(ES6), WeakMap(ES6), Set(ES6), WeakSet(ES6)

## 데이터 타입에 관한 배경지식

### 메모리와 데이터

- 0 or 1 만 표현할 수 있는 비트 단위로 위치를 확인하는 것은 비효율적이다.
- 몇 개씩 묶어 하나의 단위로 사용하면
  - 표현할 수 있는 값이 늘어난다.
  - 검색 시간을 줄 일 수 있다.
- 자바스크립트는 숫자형의 경우 **64비트(8바이트)**를 확보한다.
- 모든 데이터는 **메모리 주솟값**을 통해 서로를 구분하고 연결한다.

### 식별자와 변수

- 변수(variable)- 변할 수 있는 수, 변할 수 있는 무언가(데이터)
- 식별자(identifier) - 어떤 데이터를 식별하는 데 사용하는 이름(변수명)

### 변수 선언과 데이터 할당

```jsx
// 변할 수 있는 데이터를 만든다. 이 데이터의 식별자는 a로 한다
var a;
// 변수 a에 데이터 할당
a = 'abc';

// 변수 선언과 할당을 한 문장으로 표현
var a = 'abc';
```

- 변수란 **’변경 가능한 데이터가 담길 수 있는 공간‘** 이다.
- 변수의 영역, 데이터의 영역을 나누는 이유
  - 데이터 변환을 자유롭게 한다.
  - 메모리를 더욱 효율적으로 관리한다.
    - 데이터가 늘어날 때, 뒤에 저장된 데이터들을 전부 옮길 필요 없다.

## 기본형 데이터와 참조형 데이터

### 불변값

- 원시형
- 변수(variable)와 상수(constant)를 구분하는 성질은 ‘변경 가능성’이다.
  - 변경 가능성 - **변수 영역**
  - 불변성 - **데이터 영역**

### 가변값

- 참조형
  - 기본적인 성질은 가변값
  - 설정에 따라 변경 불가능하거나 불변값으로 활용하는 방안도 있다.
    - `Object.defineProperty`
    - `Ojbect.freeze`

## 변수 복사 비교

- 대부분의 ‘기본형은 값을 복사하고 참조형은 주솟값을 복사한다’라고 설명 하지만, 어떤 데이터 타입이든 변수에 할당하기 위해서는 주솟값을 복사해야 한다.
  - 기본형은 주솟값을 복사하는 과정을 한 번만 이뤄진다.
  - 참조형은 한 단계 더 거치게 된다.
- 기본형 데이터일 경우에는 그대로 복사하면 되지만, 참조형 데이터는 다시 그 내부의 프로퍼티들을 복사해야 한다.

## undefined와 null

- `undefined`는 사용자가 명시적으로 지정할 수도 있지만 값이 존재하지 않을 때 자바스크립트 엔지이 자동으로 부여하는 경우도 있다.
  - 값을 대입하지 않은 변수, 즉 데이터 영역의 메모리 주소를 지정하지 않은 식별자에 접근할 때
  - 객체 내부의 존재하지 않는 프로퍼티에 접근하려고 할 때
  - `return` 문이 없거나 호출되지 않는 함수의 실행 결과
- `null` 비어있음을 명시적으로 표현할 때
  - `type of null === ‘object’` 라는 점을 기억해야된다. (자바스크립트 자체 버그)
- 동등 연산자(`==`) 로 비교할때 `null`과 `undefined`가 동일하다. 일치연산자(`===`)로 비교해야 한다.
