## 클로저의 의미 및 원리 이해

- 클로저란
  - ‘**외부 스코프의 변수를 기억한 채 실행되는 함수**’
- 클로저는 함수형 프로그래밍 언어에서 등장하는 보편적인 특성이다.

```jsx
// 클로저 예시 코드
var outer = function () {
  var a = 1;
  var inner = function () {
    return ++a;
  };
  return inner;
};
var outer2 = outer();
console.log(outer2()); // 2
console.log(outer2()); // 3
```

- 클로저가 발생하는 이유
  - 어떤 값이 참조되고 있는 한, 가비지 컬렉터의 수거 대상이 되지 않는다.
  - `outer` 함수는 실행 종료 시점에 `inner` 함수를 반환한다.
  - 추후에 `inner` 함수의 실행 컨텍스트가 활성화되면 outerEnvironmentReference가 outer 함수의 LexicalEnvironment를 필요로 할 것이므로 수집 대상에서 제외한다.

- 클로저는 `return` 없이도 발생한다.
  - 외부로 전달된 함수가 지역 변수를 참조하면 클로저가 발생한다.
    - `setInterval`
    - `setTimeout`
    - `eventListener`

## 클로저와 메모리 관리

- 클로저는 메모리 누수의 위험이 있으므로 정확히 이해하고 잘 활용해야 한다.
  - 가비지 컬렉터는 도달가능성을 기준으로 수거를 판단하므로, 메모리 누수가 발생할 수 있다.
    - 루트 객체에서 시작해 도달 가능한 객체를 표시(mark)하고, 표시되지 않은 객체를 제거(sweep)하는 방식
  - 식별자에 참조형이 아닌 기본형 데이터(`null`, `undefined`)를 할당하면 된다.

## 클로저 활용 사례

- 활용 사례
  - 상태 유지
    - 클로저의 본질
  - 캡슐화
    - 정보 은닉
  - 이벤트 핸들러
    - 각 이벤트마다 고유한 상태 유지
  - 함수 팩토리
    - 특정 값을 ‘고정’한 함수를 생성

### 커링

- 하나의 함수를 여러 개의 함수로 나누어 각 단계에서 하나의 인자만 받도록 만드는 기법

```jsx
// 기본 예제
function add(a) {
  return function (b) {
    return a + b;
  };
}

add(1)(2); // 3
```

- 커링 사용 이유
  - 재사용성 증가
  - 부분 적용
  - 함수 조합
  - 가독성 및 선언적 코드 작성
- 클로저와 커링의 관계
  - 커링은 내부적으로 **클로저를 사용한 패턴**
