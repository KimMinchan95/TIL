## 배경

- ‘GoF의 디자인 패턴’ 에서 디자인 패턴
  - 공통 설계 구조의 핵심 요소를 이름짓고 추상화하여 재사용할 수 있는 객체지향 설계를 만드는데 중요한 역할을 한다.
  - 유용한 클래스와 인스턴스를 제공하여 각각의 역할과 협업, 책임의 분배를 실현한다.
- 디자인 패턴의 세 가지 유형
  - 생성 패턴
  - 구조 패턴
  - 행위 패턴

## 생성 패턴

- example
  - 생성자, 팩토리, 추상, 프로토타입, 싱글톤, 빌더
- 중점
  - 주어진 상황에 적합한 객체를 생성하는 방법에 중점을 둔다.
- 특징
  - 기본적인 객체 생성 방식은 프로젝트의 복잡성을 증가시킬 수 있어서, 이 과정을 제어하여 문제를 해결하는 것을 목표로 한다.

## 구조 패턴

- example
  - 데코레이터, 퍼사드, 플라이웨이트, 어댑터, 프록시
- 중점
  - 객체의 구성과 각 객체 간의 관계를 인식하는 방법에 중점을 둔다.
- 특징
  - 한 부분이 변경되어도 다른 부분에는 영향이 가지 않도록 도와준다.

## 행위 패턴

- example
  - 이터레이터, 중재자, 관찰자, 방문자
- 중점
  - 시스템 내의 객체 간 커뮤니케이션을 개선하거나 간소화하는 방법
- 특징
  - 객체 간의 공통적인 커뮤니케이션 패턴을 감지하고 책임을 분배함으로써 커뮤니케이션의 유연성을 높이고, 객체의 행위를 추상화한다.
