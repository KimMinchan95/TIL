## 애플리케이션 분리의 중요성

- 모듈형 JS는 애플리케이션을 모듈 단위로 쪼갤 수 있다.
  - 여러 개의 중첩된 모듈로 구성할 수 있다.
  - 느슨한 결합은 **의존성**을 낮추어 유지보수를 용이하게 만들 수 있다.

## 정적으로 모듈 가져오기

- 정적 import는 메인 코드를 실행하기 전에 모듈을 다운로드하고 실행해야 한다.

## 동적으로 모듈 가져오기

- 모듈을 초기에 모두 미리 로드하기보다 필요한 시점에만 로드하는 것이 더 이로울 때가 있다.
- 지연 로딩(Lazy-loading) 모듈을 사용하면 필요한 시점에 로드할 수 있다.
  - 사용자가 링크, 버튼을 클릭할 때 등
- 동적 import는 요청된 모듈의 네임스페이스 객체에 대한 프로미스(Promise) 객체를 반환한다.
  - `await`와도 함께 사용할 수 있다.

```jsx
form.addEventListener("submit", (e) => {
  e.preventDefault();
  import("/modules/cakeFactory.js").then((module) => {
    // 가져온 모듈 사용
  });
});
```

### 화면에 보이면 가져오기

- `IntersectionObserver` API를 사용하면 컴포넌트가 화면에 보이면 가져올 수 있다.

## 모듈을 사용하면 이점

- 한 번만 실행된다.
  - 기존 스크립트는 DOM에 추가될 때마다 실행되는 반면 모듈 스크립트는 한 번만 실행된다.
  - 의존성 트리의 가장 내부에 위치한 모듈이 먼저 실행된다.
- 유지보수와 재사용이 쉽다.
- 네임스페이스를 제공한다.
  - 개별 공간을 생성하여 글로벌 네임스페이스를 오염시키지 않는다.
- 사용하지 않는 코드를 제거한다.
  - 모듈을 통해 가져오게 되면 웹팩이나 롤업 같은 번들러를 사용해 사용하지 않는 모듈을 자동으로 제거할 수 있다.
  - 번들에 추가하기 전에 사용하지 않는 코드를 제거하는 것을 트리쉐이킹(tree-shaking)이라고 한다.
