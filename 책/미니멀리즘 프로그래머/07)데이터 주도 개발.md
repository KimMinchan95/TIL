- 코드는 주도적인 역할을 하지 않는다.
  - 코드는 우리가 가치를 전달할 때 사용하는 여러 도구 중 하나일 뿐이다.
- 데이터 구조를 활용하면 어떤 코드가 언제 실행될지 효과적으로 제어할 수 있다.
  - 이 방식은 테스트의 단숨함을 높이는 데 매우 효과적이다.
- 반복적이거나 반복에 가까운 코드를 하나로 통합해라
  - 대게 세 번 정도 반복하면 분리(데이터와 변하지 않는 코드 등) 하는 것이 좋다.

### 데이터 주도 방식의 코드는 인터프리터이다.

> 여기서의 인터프리터는 데이터 주도 방식(Data-driven)을 의미

- 인터프리터는 데이터 기반으로 명령을 받아 동작하는 코드
  - 프로그래밍 언어의 경우, 데이터는 구문 트리(synstax tree)나 바이트코드(bytecode)형태로 존재할 수 있으며, 인터프리터는 이러한 데이터를 분석해 그에 따라 동작을 수행하는 코드이다.
- 인터프리터는 소프트웨어를 유연하게 변경할 수 있도록 하는 핵심 전략이다.

### 외부 데이터에 의한 구동

- 데이터는 동적일 수 있으며, 코드의 다른 부분에서 만들어지거나 사용자의 상호작용에 따라 실시간으로 생성될 수 있다.
- 반복적인 코드가 여러 군데에서 발견된다면, 변하지 않는 부분과 변하는 부분을 분리할 수 있는지 고민해봐라.

### 테스트는 데이터 기반입니다

- 여러 테스트가 동일한 패턴을 따르면서 입력과 출력만 다르다면, 테스트 로직을 하나의 함수로 추출하고 데이터 테이블을 통해 매개변수화할 수 있다.
- 테스트가 실패할 경우, 해당 실패를 유발한 데이터를 반드시 기록하도록 해야한다.

### 상태 머신 구현하기

- 상태머신을 구현하기 위해 필요한 것은 룩업 테이블 하나뿐이다.
- 상태 머신은 이벤트(event), 상태(state), 전이(transition)으로 구성된다.
  - 이 머신은 특정 상태에서 대기하다가 이벤트가 발생하면 동작을 시작한다.
  - 이로인해 시스템이 다른 상태로 전이될 수 있다.
- 펜으로 드는 예시
  - 하나의 이벤트 유형 - 버튼이 눌렸다는 이벤트가 있다.
  - 두 가지 상태 - 심이 나온 상태와 들어간 상태가 있다.
  - 두 가지 전이가 있다 - ‘나옴 → 들어감’ 과 ‘들어감 → 나옴’ 이다.

### 기회가 있을 때마다 주의 깊게 살펴봐라

- 비슷한 코드 구문이 값만 약간씩 달라 반복된다면, 코드 구조를 리스트 기반으로 개선하는 것이 좋다.
- 일련의 동작을 순서대로 수행하지만, 그 순서가 상황에 따라 달라지는 경우에는 간단한 인터프리터로 구현할 수 있다.
- 연속적으로 발생하는 이벤트 스트림을 처리해야 하며, 이전에 발생한 이벤트가 이후 이벤트 처리에 영향을 미치는 상황이라면 상태 머신을 활용하는 방법을 검토해 볼 수 있다.
